# Dream：记忆整理流程

Dream 是 Claude Code 的后台记忆整理机制，在你不知情的情况下自动运行。

## 触发条件

三个门全部通过才触发：

```
门1 — 时间门：距离上次整理 >= 24 小时
门2 — 会话门：自上次整理后，新增会话 >= 5 个
门3 — 锁 门：当前没有其他 Dream 进程在跑
```

检查顺序按成本从低到高排列——任何一个不满足直接跳出，不做后续检查。

## 运行方式

Dream 是一个**后台分叉的子 Agent**，与主对话并行运行，互不干扰：

```
主对话（你在用）        后台 Dream Agent
      │                       │
      │  ◄──── 分叉出来 ───────┤
      │                       │
  继续响应你            读历史会话 JSONL
      │                       │
      │                  整理记忆文件
      │                       │
      │               完成后消失，记忆已更新
```

## 4 个阶段

### Phase 1 — Orient（定向）
- `ls` 记忆目录，看现有文件
- 读 `MEMORY.md` 索引，了解当前结构
- 扫已有 topic 文件，防止创建重复内容

### Phase 2 — Gather（收集）
按优先级找新信息：
1. 每日日志文件（如果有）
2. 现有记忆里已过时或矛盾的内容
3. 用 `grep` 搜索历史对话 JSONL（不全量读）

### Phase 3 — Consolidate（整合）
- 把有价值的信息写入或更新记忆文件
- 合并类似内容，不新建重复文件
- 把"昨天"、"上周"等相对时间转成绝对日期

### Phase 4 — Prune（修剪）
- 更新 `MEMORY.md` 索引
- 删掉过时的记忆指针
- 控制索引在 200 行以内、25KB 以内
- 压缩太长的条目，把细节移到 topic 文件

## 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| minHours | 24 | 触发的最小间隔时间 |
| minSessions | 5 | 触发所需的最少新增会话数 |
| SESSION_SCAN_INTERVAL_MS | 10 分钟 | 会话扫描节流间隔 |
| HOLDER_STALE_MS | 1 小时 | 锁超过此时间强制过期 |
